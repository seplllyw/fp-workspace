# 智巡 V1.9 · 版本规划总览

> 版本主题：**业务管理闭环** · 交付月份：**2026年8月** · 更新时间：2026-07-09

---

## 一、V1.9 定位

V1.8 完成了智巡的基础工具建设（地图、航线、工单、测量、小程序直播/工单）。V1.9 的核心命题是：

> **把工具串成业务闭环 — 从"能用"到"好用"**

具体来说：
1. **调度智能化**：从人工排班 → 系统按资产数据自动编排任务
2. **事件集中化**：AI 识别结果从分散在各处 → 统一事件中心收口分析
3. **能力消费化**：平台层建设的资产/设备/组织/算法能力 → 在业务端真正用起来
4. **生态可用化**：大疆妙算集成 → 航线下发+机载动作一体化

---

## 二、V1.9 功能模块（8个）

### P0（4个）— 核心闭环

| # | 模块 | 一句话 | 业务价值 |
|---|------|--------|---------|
| 1 | **智能调度** | 根据资产位置+巡检周期自动编排任务，匹配航线+设备 | 调度员审核确认即可，不用手工排班 |
| 2 | **事件管理** | AI识别/人工上报/设备告警统一收口，多维度统计+趋势分析 | 所有异常一个入口看，历史关联一目了然 |
| 3 | **算法管理（端侧）** | 异步/实时/机载三类算法在智巡端统一调用 | 巡检人员看什么问题点什么算法 |
| 4 | **生态对接** | 智巡端一键下发航线+机载动作到大疆无人机 | 规划到执行全链路打通，不切换系统 |

### P1（4个）— 支撑能力

| # | 模块 | 一句话 | 业务价值 |
|---|------|--------|---------|
| 5 | **资产管理（端侧）** | 飞行时右侧面板展示周边资产信息+历史病害 | 不用飞完再查资料 |
| 6 | **组织/人员管理** | 多组织隔离、角色权限、邀请激活 | 项目数据物理隔离，A项目看不到B项目 |
| 7 | **设备管理** | 无人机/摄像头/地面站统一注册+状态监控 | 设备在线离线一眼看清 |
| 8 | **断点续飞** | 低电量/信号中断后从断点继续 | 长距离巡检不用重飞 |

## 三、核心业务流

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 资产数据  │ →  │ 智能调度  │ →  │ 航线+设备 │ →  │ 执行巡检  │
│ (位置/周期)│    │ (自动编排)│    │ (自动匹配)│    │          │
└──────────┘    └──────────┘    └──────────┘    └─────┬────┘
                                                       │
                    ┌──────────────────────────────────┘
                    ▼
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 处置闭环  │ ←  │ 工单流转  │ ←  │ 事件中心  │ ←  │ AI 识别  │
│          │    │          │    │ (统一收口)│    │ (三类算法)│
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

**关键链路**：
- 资产 → 调度 → 航线 → 飞行 → AI识别 → 事件 → 工单 → 处置 → 关闭
- 断点续飞保障飞行连续性
- 组织权限保障数据隔离
- 设备管理保障资源可控
