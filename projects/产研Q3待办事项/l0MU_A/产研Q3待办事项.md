### V1.8 空间多维能力升级（7月交付）

#### 地图与模型管理

> 上线智巡 PC 端地图引擎与三维模型展示能力。

| 版本归属 | V1.8 |
|---------|------|
| 交付月份 | 7月 |
| 优先级 | P0 |

##### 前置依赖

- 空间服务平台 V1.0 提供三维模型数据服务
- 飞派云平台 V2.0 提供地图基础能力

##### 功能说明

构建智巡 PC 端统一地图能力，支持地图搜索（标注、设备、模型、照片检索）、多类型标注（单点、线状、面状、圆形标注）、图层与工具条（2D/3D 切换、3D 视角、正射影像显隐）。同步上线三维模型管理，支持 Mesh、点云等空间模型在地图场景中的加载与展示，为巡检场景提供高精度空间可视化底座。


##### 操作流程

1. 打开智巡 PC 端，左侧菜单进入「地图管理」，系统默认加载当前项目区域地图
2. 顶部搜索框输入设备编号、地名或标注名称，地图自动定位到目标位置
3. 工具栏选择标注类型（点/线/面/圆），在地图上点击或拖拽完成标注
4. 右侧图层面板切换 2D/3D 视角，打开正射影像叠加，对比现场实际情况
5. 切换到「三维模型」标签页，加载 Mesh 或点云模型，模型直接叠加在地图场景中

##### 核心流程

```


地图引擎加载 → 搜索定位（标注/设备/模型/照片）
    → 创建多类型标注（点/线/面/圆）
    → 图层叠加（2D↔3D切换、正射影像显隐控制）
    → 三维模型加载（Mesh/点云）→ 场景交互展示
```

![](flowcharts/智巡-地图与模型管理.png)

##### 业务价值

- 多维度地图标注能力支撑复杂业务场景
- 三维模型叠加展示增强现场感知

##### 业务目标

- 一张图看全所有巡检要素（设备、标注、模型、照片），不用来回切换系统
- 支持点、线、面、圆四种标注方式，覆盖日常巡检所有标记场景
- 三维模型直接叠在地图上展示，桥墩、边坡等复杂结构一目了然

#### 航线管理

> 支持巡逻航线全生命周期管理，实现航线资产化。

| 版本归属 | V1.8 |
|---------|------|
| 交付月份 | 7月 |
| 优先级 | P0 |

##### 前置依赖

- 地图与模型管理（地图引擎）
- 飞派云平台 V2.0 航线管理能力

##### 功能说明

提供巡逻航线的创建、编辑与管理能力，支持文件夹分类管理，航线操作涵盖复制、移动、下载、锁定及组织内分享。支持航线拼接能力，可将多条航线合并为复合航线。航线支持关联资产数据，实现"航线—资产"绑定，为后续智能调度与自动化巡检提供数据基础。


##### 操作流程

1. 在地图模块中，点击「新建航线」，输入航线名称，选择巡逻类型
2. 在地图上依次点击设置航点，右侧面板配置每个航点的飞行高度、速度、机头朝向
3. 将航线拖入对应的文件夹分类，按项目或区域归档
4. 选中航线点击「关联资产」，勾选这条航线覆盖的资产对象，完成绑定
5. 右键航线选择「分享」，设置分享范围（整个组织或指定人员）
6. 勾选多条航线，点击「拼接」，系统自动合并为一条复合航线

##### 核心流程

```


创建航线 → 参数配置（航点/高度/速度/机型）
    → 文件夹归类管理
    → 关联资产数据（资产绑定）
    → 航线操作（复制/移动/下载/锁定/分享）
    → 多航线拼接（复合航线）→ 下发至无人机
```

![](flowcharts/智巡-航线管理.png)

##### 业务价值

- 航线统一管理与复用，减少重复规划
- 航线与资产关联，巡检目标更精准
- 航线分享机制提升团队协作效率

##### 业务目标

- 巡检队长自己就能在系统里创建和编辑航线，不用每次都找技术人员
- 航线按文件夹分门别类，一个项目几十条航线也能管清楚
- 航线可以在组织内分享，A 项目踩好的航线 B 项目直接用，不用重复规划
- 多条航线拼成一条复合航线，一次飞完覆盖更大范围

#### 工单管理

> 构建完整的巡检工单流转体系。

| 版本归属 | V1.8 |
|---------|------|
| 交付月份 | 7月 |
| 优先级 | P0 |

##### 前置依赖

- 统一组织权限（飞派云平台 V2.0）

##### 功能说明

上线智巡工单管理模块，丰富工单搜索字段，支持自动创建与手工创建两种工单生成方式。工单详情关联异常信息，字段展示更加完整。支持工单转派、按组织角色流转以及完整流程记录，实现从发现异常到处置闭环的全流程跟踪。


##### 操作流程

1. 巡检过程中 AI 识别到异常，系统自动弹出工单创建窗口，预填异常截图和位置信息
2. 也可以在工单列表页点击「新建工单」，手动填写异常描述、上传照片
3. 工单自动按路段归属分配到对应养护班组，班组负责人在自己账号下看到待处理工单
4. 点击工单进入详情，看到完整的异常信息：位置、截图、AI 识别结果、历史同类工单
5. 现场处理完成后，在工单中填写处置措施、上传处理后照片，提交审核
6. 审核通过后工单自动关闭，整个流转过程的时间、人员、操作全部记录在案

##### 核心流程

```


异常触发（AI识别/人工上报）
    → 自动创建工单 / 手动创建工单
    → 工单详情（关联异常信息/位置/截图）
    → 按组织角色转派分配
    → 现场处置执行
    → 提交处置结果 → 审核关闭 → 流程记录归档
```

![](flowcharts/智巡-工单管理.png)

##### 业务价值

- 巡检异常到处置的完整闭环
- 组织角色驱动的工单流转更贴合实际业务
- 工单可追溯，便于复盘与管理

##### 业务目标

- AI 识别到异常自动生成工单，不用人工盯着屏幕逐帧找问题
- 工单按组织角色自动分配到对应负责人，谁负责哪个路段谁就收到对应工单
- 从发现异常到处理完成的每一步都有记录，出了问题能追溯到人、时间、操作

#### 测量工具

> 提供独立的测量工具，覆盖直播与地图场景。

| 版本归属 | V1.8 |
|---------|------|
| 交付月份 | 7月 |
| 优先级 | P1 |

##### 前置依赖

- 地图引擎（地图场景测量）
- 直播能力（直播场景测量）

##### 功能说明

上线独立测量工具模块，支持在直播画面与地图视图两种场景下进行距离、面积等空间测量操作。为巡检人员在远程查看现场时提供定量分析能力，辅助判定病害规模、施工范围等。


##### 操作流程

1. 在直播画面或地图视图中，点击工具栏「测量」按钮，展开测量面板
2. 选择测量类型：距离（多点连线）或面积（多边形围合）
3. 在画面上依次点击标记测量点，系统实时显示当前测量的长度或面积数值
4. 测量完成后，点击「保存」，测量结果和截图一起存入项目记录

##### 核心流程

```


选择测量场景（直播画面 / 地图视图）
    → 选择测量类型（距离 / 面积）
    → 画面中打点标记测量范围
    → 实时计算并显示结果
    → 结果保存 / 导出
```

- 远程即可完成空间测量
- 直播+地图双场景覆盖，使用更灵活

##### 业务目标

- 看直播画面就能直接量距离、算面积，不用飞到现场拿尺子量

### V1.9 业务管理闭环（8月交付）

#### 智能调度-完整版

> 基于资产能力的智能巡检调度引擎上线。

| 版本归属 | V1.9 |
|---------|------|
| 交付月份 | 8月 |
| 优先级 | P0 |

##### 前置依赖

- 航线管理（航线-资产绑定数据）
- 资产管理（资产数字平台 V1.0）
- 设备管理（设备在线状态）

##### 功能说明

新增智能调度能力，依据资产数据（位置、数字资产、巡检频率等）自动编排巡检任务，支持任务分配至合适的无人机/设备，优化巡检资源利用。结合航线与机载能力，实现从任务生成、分配、执行到结果回收的半自动化巡检闭环。


##### 操作流程

1. 进入「智能调度」，系统已按资产巡检周期自动生成本周任务建议列表
2. 调度员查看每项建议：系统推荐了哪条航线、哪架无人机、什么时间段执行
3. 确认无误后批量点击「下发」，任务自动推送到对应飞手的 APP 上
4. 在监控面板查看各任务执行进度：已下发、飞行中、已完成，异常任务标红提示

##### 核心流程

```


读取资产数据（位置/类型/巡检频率）
    → 智能编排算法（匹配航线+设备+时间窗口）
    → 自动生成巡检任务
    → 分配至无人机/设备
    → 执行巡检（航线飞行+AI识别）
    → 结果回收 → 任务完成确认 → 数据入库
```

![](flowcharts/智巡-智能调度.png)

##### 业务价值

- 巡检任务自动编排，减少人工调度
- 资源利用率提升，降低运营成本
- 为全自动化巡检奠定基础

##### 业务目标

- 系统根据资产位置和巡检周期自动排出任务计划，调度员只需审核确认，不用每天手工排班
- 任务自动匹配最近的可用无人机，资源不闲置、不冲突

#### 事件管理

> 构建巡检事件的统一管理与分析体系。

| 版本归属 | V1.9 |
|---------|------|
| 交付月份 | 8月 |
| 优先级 | P0 |

##### 功能说明

上线事件管理模块，丰富事件详情字段，支持事件多维度统计分析、趋势分析以及历史事件数据关联。用户可关注特定事件，查看关注事件的详情、趋势、图形对比与图像联动，形成事件驱动的巡检管理闭环。


##### 操作流程

1. 左侧菜单进入「事件中心」，默认按时间倒序展示所有事件，顶部可按类型/等级/时间段筛选
2. 点击任一事件查看详情：异常截图、AI 识别结果、位置标注、关联的历史同类事件
3. 对重要事件点击「关注」，后续该事件有更新自动推送通知

##### 核心流程

```


事件采集（AI识别触发/人工上报/设备告警）
    → 事件详情（类型/等级/位置/时间/关联图像）
    → 多维度统计分析（按类型/时间/区域）
    → 趋势分析（同比/环比）
    → 历史事件关联 → 关注事件追踪
    → 事件联动（图像对比/工单触发）
```

- 同类事件自动聚合，同一根因的异常不会重复出好几次，减少处置工作量

#### 算法管理（端侧应用）

> 异步/实时/机载三类算法在智巡端的统一应用。

| 版本归属 | V1.9 |
|---------|------|
| 交付月份 | 8月 |
| 优先级 | P0 |

##### 前置依赖

- AI 算法平台 V1.0（算法工程平台、场景算法模型）
- 飞派云平台 V2.0 机载能力

##### 功能说明

智巡 PC 端上线算法管理模块，将飞派云 AI 算法平台产出的异步算法、实时算法、机载算法统一接入智巡应用端。用户可在巡检流程中直接调用对应算法进行分析，结果实时回传并展示。


##### 操作流程

1. 在巡检任务中点击「算法分析」，系统根据当前场景自动推荐适用的算法列表
2. 选择算法类型：实时算法分析当前直播画面，异步算法处理已录制的视频片段
3. 对于需要机载处理的场景，点击「下发至机载」，算法指令直接推送到无人机上的妙算设备
4. 分析完成后，结果自动回传并标注在地图和视频画面上，红色框标记异常区域

##### 核心流程

```


选择巡检场景 → 选择算法类型（异步/实时/机载）
```

##### 功能说明

支持无人机在执行巡逻航线过程中，因电量不足、信号中断等原因中断飞行后，更换电池或恢复信号后从断点位置继续执行剩余航线，避免重复飞行，提升巡检效率。


##### 操作流程

1. 无人机飞行中电量低于阈值或信号中断，系统自动记录当前断点位置和剩余航线

4. 无人机自动飞到断点位置，继续执行剩余航线，完成后数据合并为一次完整飞行记录

##### 核心流程

```

    → 自动记录断点（GPS坐标/已飞航点/剩余航线）
    → 无人机返航 → 更换电池/恢复信号
    → 重新起飞 → 读取断点 → 从断点继续执行
    → 完成剩余航线 → 数据合并上报
```

![](flowcharts/智巡-断点续飞.png)

- 避免重复飞行，节省作业时间

- 长距离巡检场景实用性大幅提升

##### 业务目标

- 无人机没电了、信号断了，换完电池回来能从断点接着飞，不用重头来一遍
- 长距离巡检场景（如整条高速公路）实用性大幅提升，原来分两次飞的现在一次搞定

## 十四、Q3 重点建设方向

1. **飞行控制平台能力闭环** — 航线管理、机载动作、断点续飞、设备管理构成完整的无人机巡检管控底座，支撑多产品线复用。

2. **智能调度能力建设** — 从人工调度走向基于资产与规则的自动调度，为全自动化巡检奠定基础。

3. **AI 算法平台体系化** — 数据集→算法开发→工程化管理→端侧应用的全链路打通，覆盖 7+ 巡检场景算法模型。

4. **资产数字化能力建设** — 高速公路资产从结构化入库到 BI 分析再到端侧应用展示，形成资产数据全生命周期管理。

5. **空间服务能力建设** — 三维模型管理+空间安全+地图引擎，为上层应用提供统一的空间数据基础设施。

6. **智巡产品线深度完善** — PC 端从 V1.8 到 V2.0，小程序端从 0 到 1，覆盖地图、航线、工单、事件、智能调度、资产管理等完整业务闭环。

7. **卫瞳与消防业务能力成型** — 两条新产品线在 Q3 完成从基础能力到高级能力（态势感知、多机协作、多模态大模型）的阶梯式交付。

8. **大屏产品矩阵建立** — 智巡/卫瞳/消防/飞派云四大屏集中交付，形成覆盖全产品线的指挥监控大屏矩阵。

9. **运营中台能力建设** — 从账号与费用管理到全维度数据统计、告警工单闭环、行业数据分析，运营能力逐月递进。

10. **统一 APP 入口（FP Pilot）** — 承载智巡、卫瞳、消防三大产品线及项目定制版本，降低用户使用门槛，统一品牌入口。
