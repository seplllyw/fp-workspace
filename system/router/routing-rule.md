# Routing Rule · 路由规则

---

## 规则优先级

1. **用户指定** → 最高优先级，直接路由
2. **任务类型匹配** → 根据 task-type.md 匹配
3. **默认** → Claude（当前会话模型）

---

## 详细路由表

### MiniMax 路由

```
触发条件（满足任一）：
- 用户贴图/截图
- 关键词：截图、图片、OCR、识别、看下这个页面
- 关键词：UI、交互、按钮、布局、颜色、设计稿
- 关键词：原型、原型图、wireframe

典型任务：
- "分析这张截图"
- "这个页面有什么问题"
- "识别这张图里的文字"
- "帮我看看这个 UI"
```

### DeepSeek 路由

```
触发条件（满足任一）：
- 关键词：PRD、产品需求文档、需求
- 关键词：Roadmap、路线图、规划、版本
- 关键词：周报、汇报、月报
- 关键词：Prompt、提示词
- 关键词：Knowledge、知识库、沉淀
- 关键词：竞品、市场、调研、分析
- 关键词：Workspace 设计、架构

典型任务：
- "写一个 PRD"
- "帮我规划 Q3 Roadmap"
- "生成周报"
- "设计一个 Prompt"
- "沉淀这个知识"
- "帮我做竞品分析"
```

### Claude 路由

```
触发条件（满足任一）：
- 关键词：代码、开发、实现、bug、fix
- 关键词：重构、refactor、优化（代码层面）
- 关键词：Git、commit、分支、PR
- 关键词：Agent、Workflow、编排
- 关键词：CLI、命令、脚本
- 关键词：Workspace 整理（文件操作）
- 关键词：架构（技术架构）

典型任务：
- "帮我写一个 Python 脚本"
- "重构这段代码"
- "创建一个 Workflow"
- "提交代码"
- "整理 Workspace"
```

---

## 复合任务处理

```
示例："分析这个页面截图，然后写 PRD"
  → 拆分为：
    1. 图片分析 → MiniMax
    2. PRD 撰写 → DeepSeek

示例："根据需求，开发一个HTML原型"
  → 拆分为：
    1. 需求分析 → DeepSeek（可选，复杂需求时）
    2. 代码实现 → Claude
```
