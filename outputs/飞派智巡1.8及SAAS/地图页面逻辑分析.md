## 二、导航与路由系统

### 2.1 两级导航架构

```
一级 Tab (顶部)          二级侧边栏组               页面 (Page Panel)
─────────────────────────────────────────────────────────────
工作台 (workbench)  →  workbench 侧边栏组  →  整体态势(situation) / 多路直播(live) / 应急飞行(emergency) / 巡检场景(inspection)
数据管理 (data-mgr)  →  data-mgr 侧边栏组   →  模型库 / 地图管理 / 航线管理 / 资产管理 / 媒体管理
工作流 (workflow)    →  workflow 侧边栏组   →  飞行任务 / 事件管理 / 工单管理
算法仓库 (algorithm) →  algorithm 侧边栏组  →  算法仓库（无二级 tab）
资源管理 (resource)  →  resource 侧边栏组   →  人员管理 / 设备管理 / 空域管理
```

### 2.2 配置驱动路由

```javascript
const topTabConfig = {
  'workbench': { defaultPage: 'situation', sidebarGroup: 'workbench' },
  'data-mgr':  { defaultPage: 'model-lib', sidebarGroup: 'data-mgr' },
  'workflow':  { defaultPage: 'flight-task', sidebarGroup: 'workflow' },
  'algorithm': { defaultPage: 'algorithm', sidebarGroup: 'algorithm' },
  'resource':  { defaultPage: 'personnel-mgr', sidebarGroup: 'resource' },
};
```

### 2.3 切换流程

```
switchTopTab(topTabId)
  1. 防止重复切换（currentTopTab === topTabId → return）
  2. 更新顶部 Tab active 态
  3. 切换 sidebar 组显示（隐藏所有 .sidebar-tab-group，显示匹配组）
  4. 调用 switchPage(cfg.defaultPage) 切换到默认页面
  5. 关闭巡检下拉

switchPage(pageId)
  1. 更新侧边栏 Tab active 态（在当前可见 sidebar 组内查找）
  2. 特殊处理：巡检场景激活父级 tab
  3. 切换 page panel 显示（隐藏所有 .page-panel，显示匹配页）
  4. 关闭巡检下拉
```

### 2.4 巡检场景下拉菜单

- 点击"巡检场景"tab → toggleDropdown() 切换展开/收起
- 下拉菜单定位：`left: sidebar-w + 4px`（侧边栏右侧弹出）
- 点击外部区域 → 自动关闭
- 选择子项（路面巡检/交安设施巡检/交通巡检）→ 切换到 inspection page + 更新占位内容

## 四、地图初始化流程

```
initMap()
  ├─ L.map('map', { center: [30.263, 120.123], zoom: 14, zoomControl: false, attributionControl: false })
  ├─ 添加 CSS class 'mode-satellite' 到 map-container
  ├─ tileLayers.satellite.addTo(map)              ← 默认卫星底图
  ├─ featureGroups.asset.addTo(map)                ← 资产图层
  ├─ featureGroups.device.addTo(map)               ← 设备图层
  ├─ featureGroups.model.addTo(map)                ← 模型图层（空）
  ├─ featureGroups.airspace = L.featureGroup().addTo(map)  ← 空域图层
  ├─ flightRouteGroup = L.featureGroup().addTo(map) ← 航线图层
  ├─ 绑定事件：map click → handleDrawClick / handleMeasureClick
  ├─ 绑定事件：map dblclick → handleDrawDblClick
  ├─ 绑定事件：map mousemove → handleDrawMouseMove + updateMouseCoords
  ├─ 绑定事件：map moveend / zoomend → simulateDataLoad
  └─ renderAll()

renderAll()
  ├─ renderAssets()          ← 渲染资产标记
  ├─ renderDevices()         ← 渲染设备标记
  ├─ renderAirspace()        ← 渲染空域区域
  └─ renderDevicePanel()     ← 渲染底部设备栏按钮

后续调用：
  ├─ initToolbarMapBindings()  ← 工具栏事件 + 键盘
  ├─ initOrthoDefaults()       ← 初始化正射影像 _enabled 和 bounds
  ├─ renderFlightRoutes()      ← 渲染飞行航线
  ├─ startFlightAnimation()    ← 启动无人机飞行动画
  └─ lucide.createIcons()      ← 初始化图标
```

### 4.1 底图瓦片层

```javascript
const tileLayers = {
  satellite: L.tileLayer('https://server.arcgisonline.com/...World_Imagery/...'),
  ortho:     L.tileLayer('https://server.arcgisonline.com/...World_Imagery/...'),

// 两个图层使用相同的 URL，区别在于 CSS 滤镜

底图切换通过 `applyMapMode(modeClass, tileKey, modeName, noticeText)` 实现：
1. 移除旧 CSS class（mode-satellite / mode-ortho）
2. 切换 Leaflet tile layer
3. 添加新模式 CSS class → 触发 CSS 滤镜
4. 更新图层单选框状态
5. 显示模式指示器浮层（1.8 秒后消失）

## 五、资产系统逻辑

### 5.1 资产渲染（renderAssets）

```
renderAssets()
  for each asset in MOCK.assets:
    if asset._enabled === false → skip    ← toggle 控制可见性
    
    确定 marker CSS class：
      status === 'abnormal' → 'marker-task-error'（红色）
      status === 'offline'  → 'marker-task-pending'（灰色）
      其他                   → 无额外 class（紫色默认）
    
    创建 L.divIcon({ className: 'custom-marker marker-asset', html: landmark图标 })
    iconSize: [28, 28], iconAnchor: [14, 14]
    
    创建 L.marker → 绑定事件 → 添加到 featureGroups.asset
```

### 5.2 资产面板（Expandable Panel）

```
toggleAssetPanel()  ← 点击"资产"行触发
  if 正射影像面板打开 → 先关闭
  切换 ortho-expandable "open" class → 箭头旋转 90°
  切换 ortho-panel "open" class → max-height: 0 → 200px 展开动画
  
  renderAssetList(filterText?)
    筛选：name / id / desc 包含搜索关键词
    每个资产行：
      - 状态圆点（绿/红/灰）
      - 名称 + ID + 状态文字
      - Toggle 开关（联动 _enabled 属性）

toggleAssetMarker(id, enabled)
  更新 asset._enabled
  重新 renderAssets()          ← 全量重渲染
  重新 renderAssetList()       ← 刷新列表 UI
```

### 5.3 资产搜索

```
asset-search input → input 事件 → renderAssetList(this.value)
  → 对 MOCK.assets 做字符串模糊匹配
  → 仅过滤列表显示，不改变地图渲染
```

## 六、设备系统逻辑

### 6.1 设备渲染（renderDevices）

```
renderDevices()
  清除旧的信息标签（_infoTag 引用）
  featureGroups.device.clearLayers()
  droneMarkersMap = {}

  for each device in MOCK.devices:
    确定 marker CSS class：
      type === 'station'           → 'marker-station'（青色）
      status === 'flying'          → 'marker-flying'（蓝色）
      status === 'standby'         → 'marker-standby'（绿色）
      其他                         → 'marker-task-pending'（灰色）

    创建 DivIcon：
      基站 → antenna 图标
      无人机 → plane 图标

      引用存入 device._infoTag

## 九、飞行航线与动画系统

### 9.1 航线渲染（renderFlightRoutes）

```
for each route in MOCK.flightRoutes:
  1. 绘制虚线航线
     L.polyline(waypoints, { color, weight: 3, dashArray: '10 6', opacity: 0.7 })
  
  2. 绘制航点圆圈
     L.circleMarker(wp, { radius: i===0 ? 5 : 3 })  ← 起点稍大
  
  3. 航线名称标签
     在路径中点放置 route-label-badge
  
  4. 进度线（初始为空）
     L.polyline([], { color, weight: 4, opacity: 0.9 })
     → route._progLine
```

### 9.2 飞行动画（startFlightAnimation）

```
tick() 循环（50ms 间隔 = 20fps）：
  for each route in MOCK.flightRoutes:
    // 1. 推进进度
    state.progress += 0.0005
    if progress > 1 → reset to 0（循环飞行）

    // 2. 线性插值获取当前位置
    pos = interpolatePosition(route, progress)
    
    // 3. 计算 bearing（用于旋转无人机图标）
    bearing = calculateBearing(current, nextPosition)
    
    // 4. 更新无人机数据
    drone.lat = pos[0], drone.lng = pos[1]

    // 5. 更新地图上的无人机标记
    updateDroneMarker(drone, bearing)
      → marker.setLatLng([lat, lng])
      → 旋转 CSS transform: rotate(bearing deg)
      → 重新创建 Live 预览卡片（带最新 alt/speed/battery）
    
    // 6. 更新进度线
    updateProgressLine(route, progress)
      → 计算已飞路径的 partial latLngs
      → _progLine.setLatLngs(partial)

停止条件：设备 toggle 关闭 → stopFlightAnimation()
```

### 9.3 位置插值算法

```
interpolatePosition(route, progress):
  1. 计算各段距离 → segs[{from, to, dist}]
  2. 总距离 totalDist
  3. target = progress * totalDist
  4. 累加各段距离，找到 target 所在段
  5. 在该段内线性插值：t = (target - acc) / seg.dist
  6. 返回 [lat1 + (lat2-lat1)*t, lng1 + (lng2-lng1)*t]
```

### 9.4 Bearing 计算

```
calculateBearing(lat1, lng1, lat2, lng2):
  使用球面几何公式
  返回 0-360 度（正北为 0）
  用于旋转无人机图标的 CSS transform
```

### 10.3 面积计算（球面几何）

```javascript
// Polygon 面积（使用球面三角形面积公式）
var area = 0, R = 6371000, n = drawPts.length;
for (var i = 0; i < n; i++) {
  var j = (i+1) % n;
  area += (drawPts[j][1] - drawPts[i][1]) * Math.PI/180
        * (2 + Math.sin(drawPts[i][0]*Math.PI/180) + Math.sin(drawPts[j][0]*Math.PI/180));
}
area = Math.abs(area * R * R / 2);  // 单位：m²
```

## 十九、核心设计模式总结

1. **全量重渲染**：大部分 toggle/编辑操作触发全量 `render*()` 而非增量更新，简单可靠
2. **运行时属性**：`_enabled`, `_overlay`, `_infoTag` 等以下划线开头的属性是运行时注入的，不属于数据模型本身
3. **Layer 引用缓存**：标记通过 `droneMarkersMap[id]`、`_progLine`、`_infoTag` 等方式缓存 Leaflet Layer 引用，便于动画更新
4. **互斥面板**：资产面板和正射影像面板互斥（打开一个关闭另一个）
5. **事件委托**：图层面板使用事件委托 + `closest()` 查找触发元素，避免为每个子元素单独绑定
6. **CSS 驱动模式切换**：地图底图模式通过 CSS class 控制滤镜，而非替换瓦片 URL
7. **绘制状态机**：标注系统使用 `drawTool` 状态变量 + 阶段判断（如 circleCtr）实现多阶段交互
8. **配置驱动 UI**：顶栏 tab 配置、巡检场景子页面信息、标注的图标/颜色/字体大小等都可配置
